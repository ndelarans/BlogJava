JavaScript Asíncrono

El JavaScript asíncrono permite que el código continúe ejecutándose mientras se espera una respuesta de una operación. Esto significa que no bloquea la ejecución de otras funciones, lo que es esencial para mejorar la experiencia del usuario al realizar múltiples tareas al mismo tiempo.

Las principales herramientas para la programación asíncrona en JavaScript son las promesas, los callbacks y las funciones asíncronas/await. Estas técnicas permiten manejar las tareas largas de forma eficiente, como la carga de datos desde servidores o la interacción con APIs.

Ejemplo de Código Asíncrono con async/await


            // Función asíncrona que obtiene datos de una API
            async function fetchData() {
                try {
                    const response = await fetch('https://api.example.com/data');
                    const data = await response.json();
                    console.log(data);
                } catch (error) {
                    console.error('Error:', error);
                }
            }
            
            fetchData();

Callback

Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que se complete una operación. Los callbacks son fundamentales en JavaScript para manejar operaciones asíncronas, como solicitudes a servidores, manejo de eventos o temporizadores.

Por ejemplo, en operaciones que requieren tiempo, como la obtención de datos de una API, puedes pasar una función callback para que procese los datos cuando estén listos. Así, el código puede seguir ejecutándose sin esperar a que finalice la operación asíncrona, mejorando el rendimiento de la aplicación. 


                    function myCallbackFunction(result) {
                        console.log('Resultado:', result);
                    }
                    
                    function processData(callback) {
                        const data = "Hola, este es el dato procesado.";
                        callback(data);
                    }
                    
                    processData(myCallbackFunction);

Promise

Una Promise en JavaScript es un objeto que representa la eventual finalización (o fracaso) de una operación asíncrona y su valor resultante. Es fundamental para manejar operaciones que toman tiempo, como llamadas a APIs o temporizadores.

Las promesas tienen tres estados posibles:

  • Pendiente (Pending): cuando la operación está en curso.
  • Resuelta (Fulfilled): cuando la operación ha sido completada con éxito.
  • Rechazada (Rejected): cuando la operación ha fallado.
Las promesas permiten estructurar el código de manera que las operaciones asíncronas sean más fáciles de manejar y evitar el uso excesivo de callbacks anidados, conocido como "callback hell". 


                function fetchData() {
                    return new Promise((resolve, reject) => {
                        setTimeout(() => {
                            const success = true;
                            if (success) {
                                resolve("Datos obtenidos con éxito.");
                            } else {
                                reject("Error al obtener los datos.");
                            }
                        }, 2000);
                    });
                }
        
                fetchData()
                    .then(result => console.log(result))
                    .catch(error => console.error(error));

Async-Await

Async-Await es una forma más sencilla de trabajar con promesas en JavaScript. Se utiliza para manejar operaciones asíncronas, permitiendo escribir código que parece síncrono mientras espera la resolución de una promesa. La palabra clave `async` indica que una función es asíncrona, y `await` se usa dentro de estas funciones para esperar a que la promesa se resuelva. 


                    async function fetchData() {
                        try {
                            const response = await fetch('https://api.example.com/data');
                            const data = await response.json();
                            console.log(data);
                        } catch (error) {
                            console.error('Error:', error);
                        }
                    }
        
                    fetchData();

Event Loop

El Event Loop es un mecanismo de JavaScript que maneja la ejecución de código, recogida de eventos y tareas asíncronas (como las promesas y los callbacks). El Event Loop verifica continuamente la pila de ejecución y la cola de tareas, asegurando que las operaciones asíncronas se manejen correctamente. 


                    console.log('Inicio');
        
                    setTimeout(() => {
                        console.log('Dentro de setTimeout');
                    }, 1000);
        
                    console.log('Fin');
